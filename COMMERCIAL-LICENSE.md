## When you need a commercial license

The following components are licensed under ELv2:

- `aiss/` — Core engine (identity, chains, crypto, memory)
- `vigil/` — Behavioral monitoring (TS, TSI, A2C, VRS)
- `trustgate/` — Governance engine (policy, REQUIRE_HUMAN, compliance)

ELv2 prohibits one specific use case:

> **Providing the software to third parties as a hosted or managed
> service**, where the service provides users with access to any
> substantial set of the features or functionality of the software.

You need a commercial license if you:

- Offer PiQrypt as a managed service or hosted product to customers
- Embed PiQrypt core/Vigil/TrustGate in a SaaS product sold to third parties
- Deploy as a cloud service (AWS, GCP, Azure, or private cloud) for external users
- Cannot accept ELv2 in your organization's legal framework
- Want to keep modifications to the core private without any open-source obligation

You do **not** need a commercial license if you:

- Use PiQrypt internally (no external users interact with it)
- Build an open-source project (apply for OSS license instead)
- Use only the bridges, CLI, or AISS specification (MIT/Apache-2.0)
- Are an individual developer or researcher

## Relationship with subscription tiers

The **commercial license** and the **subscription tiers** are independent:

| | Subscription (Pro/Business/Enterprise) | Commercial License |
|---|---|---|
| **What it covers** | Feature access (quota, Vigil full, TrustGate full) | IP rights (deploy as SaaS, keep code private) |
| **Who needs it** | Any user who wants Pro features | Organizations deploying as managed service |

A typical enterprise customer holds:
- A **Business or Enterprise subscription** for feature access

## FAQ

**Can we evaluate before signing a commercial license?**
Yes. ELv2 allows full use for internal evaluation, development,
and testing — only external deployment to third parties requires
a commercial license.

**We are a system integrator building for a client — do we need a license?**
If you deploy PiQrypt in an environment where your client's users
interact with it, yes. Contact us — SI partner terms are available.

**We already have a Pro/Business subscription — do we still need a commercial license?**
If you are deploying PiQrypt as a service to your own customers
(not just using it internally), yes — the subscription covers
feature access, not SaaS deployment rights.

**What happens if we grow past the startup threshold?**
Startup licenses include a grace period and upgrade path.
Contact us before you hit $1M ARR and we will transition you smoothly.

**Is the commercial license perpetual?**
Licenses are annual by default. Perpetual licenses available
for OEM and embedded deployments — contact us.
